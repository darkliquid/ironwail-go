PARITY TARGET
=============

  PRIMARY:    CGO/OpenGL path  (renderer_opengl.go + GLFW + SDL3/Oto)
  SECONDARY:  gogpu/WebGPU     (NOT a parity gate; intentionally behind)

  Do not judge parity from the gogpu path.
  Run: mise run build-cgo  and  mise run smoke-cgo-map-start  for OpenGL checks.


CURRENT STRENGTHS (OpenGL path)
================================

  - World rendering: BSP geometry, lightmaps, animated textures, turbulent UV
    warp, fog, embedded BSP sky, external skybox (cubemap + per-face paths)
  - Entity rendering: alias models, sprites, brush submodels, particles,
    decals, dynamic lights, viewmodel, all with correct pass ordering
  - Input / bindings: Quake-style bind/unbind/bindlist, config persistence
  - Console: tab completion (commands/cvars/aliases), history, scrollback
  - Menus: main, load/save, options (video/audio/controls), multiplayer setup,
    bounded join/host flows
  - HUD: classic status bar, intermission/finale overlays, deathmatch scoreboard
  - Audio: sound dispatch, mixer, spatialization, WAV/OGG CD tracks, ambient
  - Save/load: full QC/edict/global round-trip, C-style save restrictions
  - Demo: forward record/playback including connected-state snapshot
  - Networking: loopback + UDP; remote connect/disconnect/reconnect/kick


BIGGEST REMAINING GAPS
======================

  1. gogpu backend        Full entity/model/sprite pipeline is a stub
  2. OpenGL fidelity      Broader visual polish beyond bounded slices
  3. Mods menu            No Ironwail-style installed add-on browser
  4. Alt HUD styles       Q64-style HUD variants not implemented
  5. Mouse-driven UI      Menu mouse interaction is incomplete
  6. Remote netgame       Deeper multiplayer / net-game behavior still needed
  7. Audio codec parity   Broader music format coverage beyond WAV/OGG
  8. Demo long-tail       Edge-case recording/playback accuracy
  9. Save UX long-tail    Broader save-file search, legacy import edge cases
 10. Underwater fx        Visual blue-shift parity open (audio intensity wired)
