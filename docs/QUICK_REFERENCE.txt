COMMON PITFALLS TO AVOID
========================

1. Don't render entities before world geometry (depth test will fail)
2. Lightmap coordinates are NOT texture coordinates (different calculation)
3. Prediction must accumulate commands, not just apply current frame
4. Sprite rotation requires full 3D camera basis vectors
5. Audio attenuation uses both distance AND listener angle (3D panning)
6. Menu draws OVER 3D view, not instead of it
7. Sky must render FIRST (before world geometry), regardless of depth

GOTCHAS IN GO CONVERSION
========================

- C array bounds check missing: bounds = []byte{0x6f, 0x6d, ...} is OK
- Entity lerp flags use bitwise: state & 1, state |= 2 (use bit operators)
- Light cache is sparse: allocate on-demand, not full array
- Particle colors are palette indices: use ramp lookup, not RGB directly
- Console commands are case-insensitive: normalize before dispatch
- CVars can have callbacks: need event system when value changes
- Skin textures are per-player: dynamic allocation for scoreboard
